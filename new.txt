我建议的前端方案
一、先做整体布局重构
你现在 MainWindow.xaml 结构很简单：

外层 Grid
一个背景 Border
一个 WebView2
一个状态提示 Border
这不够支撑真实产品界面。
建议改成下面这种三段式：

建议结构
顶部工具栏：标题、打开文件、设置、缩放、主题切换
中间阅读内容区：WebView2
角落状态区：加载状态、页码、章节、网络/渲染状态
这样按钮就不会和正文抢位置。

二、把“顶部白边”彻底消掉
白边通常来自以下几类问题：

可能来源
WindowStyle=None + 自定义圆角容器 + 根背景不一致
Border 的 CornerRadius 让边缘露出底层窗口背景
WebView2 自身背景透明度和宿主窗口背景不一致
系统窗口边框或 DPI 缩放导致边缘留白
解决方案
窗口根背景和内容背景统一
最外层 Grid、内层 Border、WebView2 页面背景都设成一致颜色
不要让透明背景直接露底
如果用圆角，外层容器必须完整包住窗口，不要裸露边缘
推荐颜色统一
比如统一成：

窗口底色：#F7F3EA
内容卡片：#FFFDF8
辅助背景：#EDE3D3
文字主色：#2A2521
次级文字：#6B5D50
这样白边会显得像“设计留白”，而不是“没处理干净”。

三、按钮不要和内容抢空间
现在按钮遮挡的问题，本质上是按钮没进入明确的布局流。

建议
把按钮固定在顶部工具栏，不要漂在内容上面。
如果一定要悬浮，必须有：

固定宽高
半透明底
足够内边距
不影响正文区域
必要时自动收起
推荐按钮排布
从左到右：

打开
最近阅读
目录
设置
缩放减
缩放重置
缩放加
主题切换
窗口模式切换
不要把太多按钮堆成一排，容易挤压。
如果功能多，就分成“主操作”和“次操作”。

四、把阅读正文区做成真正的“阅读版式”
这是美化里最重要的一部分。

现在的问题
当前正文区更像一个测试页面，而不是阅读器。

建议优化
正文居中

设定最大阅读宽度，比如 680px ~ 820px
大屏下不要无限拉宽

注释：略小
章节标题：有明显间距
行距

正文建议 1.7 ~ 2.0
不要太密
段间距

段落之间保留适当空白
让阅读更轻松
目录栏

目录栏固定宽度但可折叠
小窗口下自动隐藏或缩窄
滚动条

细一点，颜色弱一点
不要太抢眼

六、前端层级要统一

五、白边问题的具体处理方案
顶部白边通常不是一个地方造成的，往往是几个层叠在一起。

1. 外层窗口背景统一
窗口最外层和内层 Border 颜色必须一致，不能一个白一个米色。

建议
Window.Background = "#F7F3EA"
最外层容器背景也用同色
WebView2 页面 body 背景也用同色
2. 圆角边缘不要“露底”
如果用了 CornerRadius，边缘会显示底层内容。
底层内容如果是白色，就会看见白边。

解决
外层容器必须有完整背景
不要让透明区域直接露出系统默认底色
Web 页面也要补同色背景
3. WebView2 页面 body 要去掉默认边距
你现在 HTML 里已经有：

html, body { margin: 0; min-height: 100%; background: transparent; }
这里建议改成非透明背景，直接设置为统一底色。
透明背景很容易把底层白色漏出来。

更稳的思路
body 直接用页面底色
不依赖透明叠加
阅读区自己居中留白
六、推荐的交互方案

2. 左侧目录可折叠

3. 悬浮提示自动消失

这些提示不要固定占位。

4. 快捷键和按钮并存
按钮负责发现性，快捷键负责效率。
